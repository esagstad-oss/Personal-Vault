# YouTube Transcription Workflow

When the user asks to transcribe a YouTube video, use the existing script — do not build a new one.

## Script
```
C:\GitRepo\Personal\Youtube Transcriptions\transcribe_youtube.py
```
Requires: `youtube-transcript-api` (already installed).

## Steps
1. Create a subfolder: `C:\GitRepo\Personal\Youtube Transcriptions\<Channel> - <Topic>\`
2. Run the script from that subfolder so the `.txt` output lands there:
   ```powershell
   Set-Location "C:\GitRepo\Personal\Youtube Transcriptions\<Channel> - <Topic>"
   python "..\transcribe_youtube.py" "<youtube_url>"
   ```
3. Read the full transcript `.txt` file.
4. Analyse the content and generate `transcript_<videoId>_highlights.html` in the **same subfolder**.
   - Self-contained dark-themed HTML (no external dependencies)
   - Always include these sections:
     - **Key Facts at a Glance** — pill-style stat badges
     - **Topic deep-dives** — one card per major concept or theme
     - **Notable Quotes** — blockquote styled
     - **Actionable Takeaways / Top Tips** — numbered list with counter badges
     - **Red Flags** (if health/medical topic) — warning-styled items
   - Use colour-coded cards, tables, pill badges for readability
5. Open the HTML with `Start-Process`.

## Folder structure
```
Youtube Transcriptions/
├── transcribe_youtube.py             ← do not move
└── <Channel> - <Topic>/
    ├── transcript_<videoId>.txt
    └── transcript_<videoId>_highlights.html
```

## API note
The installed version uses an instance-based API:
```python
api = YouTubeTranscriptApi()
transcript_list = api.list(video_id)
transcript = transcript_list.find_transcript(["en"])
fetched = transcript.fetch()
```
